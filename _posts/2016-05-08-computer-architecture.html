The fetch-execute cycle

The CPU reads instructions from memory and then executes them. Memory is composed of numbered addresses, each of which point to a storage location. Data and programs are stored in memory. The CPU performs these operations one instruction at a time. In order to be able to access and change data, the CPU uses the following elements:

  • Program Counter: tells the computer where to get the next instructions
  • Instruction Decoder: figures out what the instructions mean
  • Data bus: the connection between the CPU and memory
  • General purpose registers: memory location on the CPU itself
  • Arithmetic and Logic unit: actually executes the instruction

Basics of Computer Memory

  • Address: the number attached to a memory storage location
  • Byte: smallest addressable unit for a CPU. de facto standard of 8 bits. For x86 processes, its a number between 0 and 256
  • Word size: the typical size of a register
  • Pointer: an address stored in memory. Points to a different address

Addressing modes

Different ways to access data

  • Immediate: data to access is embedded in the instruction itself
  • Direct: instruction contains an address to load data from
  • Indexed: instruction contains an address and an offset
  • Indirect: instructions contain a register that contains a pointer
  • Base-pointer: like indirect, but also includes an offset

Registers

Registers are memory locations that live in the CPU itself. Because the registers are actually inside the CPU itself, operations involving registers are fast. When the CPU is performing calculations or manipulating data, the registers are where data is temporarily stored. These registers all have names and some of them have special uses and purposes. For example, the instruction pointer is a special register that contains the address of the next instruction to be fetched from memory and executed.
